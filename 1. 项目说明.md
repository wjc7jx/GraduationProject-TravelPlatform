# 📘 旅行项目管理系统（微信小程序 + Node.js）
**Graduation Project Base Document**

---

## 一、项目概述
### 1. 项目名称
旅行项目管理系统（Travel Project Management System）

### 2. 项目定位
本系统是一个面向个人用户的**旅行记录与回溯平台**，通过整合图片、文字、音频、GPS轨迹等多媒体数据，以“时间轴 + 地图”的方式构建沉浸式旅行故事。

### 3. 项目目标
+ 帮助用户结构化记录旅行过程
+ 实现多媒体内容的统一管理与展示
+ 提供可视化的“故事地图”体验
+ 支持导出与分享旅行纪念内容
+ 保障用户隐私与数据安全

---

## 二、技术选型（当前基准）
### 1. 前端
+ 微信小程序（原生 / uni-app 可选）
+ 地图服务：高德地图 / 腾讯地图
+ 富文本编辑器（自定义或第三方）

### 2. 后端
+ Node.js + Express 
+ ORM：Sequelize
+ 数据库：MySQL
+ 鉴权：JWT
+ 文件存储：本地存储 / OSS（可扩展）

### 3. 多媒体处理
+ 图片：EXIF解析（时间/GPS）
+ 轨迹：GPX / KML 简化解析
+ 音频：上传 + 简单播放控制

### 4. 导出能力
+ PDF生成：pdfkit
+ HTML导出：静态页面生成

## 三、核心功能模块
---

### 1. 旅行项目管理模块（用户端）
#### 功能描述
用户可以创建和管理多个旅行项目。

#### 功能点
+ 创建旅行项目
    - 封面图片
    - 标题
    - 起止时间
    - 标签（如：自驾 / 出境 / 徒步）
+ 项目管理
    - 编辑项目
    - 删除项目
    - 项目归档（历史记录）

---

### 2. 多媒体内容聚合模块
#### 功能描述
统一管理旅行中的多种内容，并进行时间与空间关联。

#### 功能点
##### （1）照片上传
+ 从手机相册选择图片
+ 自动解析 EXIF 信息：
    - 拍摄时间
    - GPS位置（若存在）

##### （2）文字日记
+ 富文本编辑（支持图文混排）
+ 自动绑定时间戳
+ 可插入图片

##### （3）GPS轨迹记录
+ 支持模拟轨迹记录（小程序后台限制）
+ 支持导入文件：
    - GPX
    - KML（简化解析）

##### （4）音频片段
+ 支持录制或上传音频
+ 限制时长 ≤ 60 秒
+ 支持播放

##### （5）内容组织
+ 按时间自动排序
+ 支持手动排序调整
+ 支持绑定地图位置

---

### 3. 故事地图视图模块（核心亮点）
#### 功能描述
实现“时间 + 空间”双维度展示旅行内容。

#### 界面结构
+ 左侧：时间轴（Timeline）
    - 按天分组
    - 展示事件节点（缩略图 + 标题）
+ 右侧：地图（Map）
    - 标注所有带GPS的数据点
    - 点击标记查看详情

#### 核心能力
+ 时间轴与地图联动
+ 地图交互：
    - 缩放
    - 拖拽
+ 时间筛选（按日期过滤）

#### 用户体验
实现：

📍 看地图 + 📝 读故事 + 🔊 听声音

---

### 4. 导出与分享模块
#### 功能描述
将旅行内容生成可传播的成果。

#### 功能点
##### （1）PDF导出
+ A4排版
+ 包含：
    - 封面
    - 目录
    - 图文内容

##### （2）网页纪念册
+ 生成单页HTML
+ 内联CSS/JS
+ 可：
    - 本地打开
    - 部署到网站

##### （3）数据控制
+ 仅导出用户授权内容

---

### 5. 隐私控制机制
#### 功能描述
对用户数据进行分级保护。

#### 可见性级别
| 类型 | 说明 |
| --- | --- |
| 私密 | 仅用户本人可见 |
| 好友可见 | 白名单机制（模拟微信好友） |
| 公开 | 可被推荐或搜索 |


#### 隐私保护
+ 地理位置模糊处理（如家庭住址）
+ 敏感数据过滤

+ 项目 1:N 内容
