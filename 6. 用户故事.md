# 旅行项目管理系统 - 用户故事（User Stories）

本项目采用敏捷开发中的用户故事格式（As a `<Role>`, I want to `<Action>`, so that `<Benefit>`）对《1. 项目说明》中的需求进行拆解，以便于指导后续的数据库完善、后端API开发及小程序页面开发。

---

## Epic 1：账号与基础设置
**面向用户：普通旅行者**

### 1.1 微信快速登录
* **用户故事**：作为一个旅行者，我想要通过微信一键授权登录系统，以便于我能快速开始记录旅行，而不需要繁琐的注册流程。
* **验收标准**：
  - [ ] 首次打开小程序时提示微信授权。
  - [ ] 后端通过 WeChat OpenID 自动创建或关联用户记录。
  - [ ] 登录成功后发放 JWT Token 用于后续接口鉴权。

## Epic 2：旅行项目管理（Project）
**针对核心模块 1**

### 2.1 创建旅行项目
* **用户故事**：作为一个旅行者，我想要创建一个包含封面、标题、起止时间和标签的“旅行项目”，以便于为单次旅行建立一个独立的归档文件夹。
* **验收标准**：
  - [ ] 支持上传或选择一张图片作为项目封面。
  - [ ] 支持输入项目标题（必填）和选择起止日期范围。
  - [ ] 提交后在数据库生成新的 Project 记录。

### 2.2 项目列表概览
* **用户故事**：作为一个旅行者，我想要在小程序首页看到我所有的历史和正在进行中的旅行项目，以便于我随时回顾或继续添加内容。
* **验收标准**：
  - [ ] 首页以卡片列表形式展示所有项目。
  - [ ] 卡片展示封面、标题、日期范围。
  - [ ] 支持侧滑或长按进行“编辑”或“删除/归档”操作。

## Epic 3：多媒体内容聚合（Content）
**针对核心模块 2**

### 3.1 极速照片上传与解析
* **用户故事**：作为一个旅行者，我想要上传旅行照片时，系统能自动提取照片的拍摄时间和GPS位置，以便于我无需手动输入即可将照片对齐到时间轴和地图上。
* **验收标准**：
  - [ ] 小程序端选择图片后上传到服务器或OSS。
  - [ ] 后端读取图片的 EXIF 数据，提取 DateTimeOriginal 和 GPSLatitude/GPSLongitude。
  - [ ] 自动生成一条 Content（类型为图片），并关联对应的 Location 记录。

### 3.2 编写富文本图文日记
* **用户故事**：作为一个旅行者，我想要编写一篇图文并茂的富文本日记，并为其指定特定时间，以便于详细记录我当时的旅行感悟。
* **验收标准**：
  - [ ] 提供富文本编辑器（支持加粗、标题、插入图文）。
  - [ ] 默认绑定当前时间，但允许用户手动修改时间戳。

### 3.3 记录语音随笔
* **用户故事**：作为一个旅行者，我想要直接录制一段不超过60秒的语音并上传，以便于快速记录当时的环境声音或口述心情。
* **验收标准**：
  - [ ] 小程序端支持长按录制或点击录制，限制60秒。
  - [ ] 界面上展示语音条和播放/暂停控制按钮。

### 3.4 导入徒步或自驾轨迹
* **用户故事**：作为一个旅行者，我想要导入包含位置点的 GPX 或 KML 文件，以便于系统能还原我当天的完整行进路线。
* **验收标准**：
  - [ ] 后端支持解析 GPX/KML 节点，简化抽稀处理后存入数据库。
  - [ ] 在地图上能以连线的形式（Polyline）绘制出这段路线。

## Epic 4：故事地图视图联动（亮点与难点所在）
**针对核心模块 3**

### 4.1 双维度的沉浸式浏览
* **用户故事**：作为一个旅行者，我想要在同一个界面中，左边看按时间排序的事件时间轴，右边看打满地理坐标点的高德/腾讯地图，以便于获得空间和时间交织的回忆体验。
* **验收标准**：
  - [ ] 界面分为Timeline组件和Map组件。
  - [ ] Timeline按天分组展示事件的文字缩略和缩略图。
  - [ ] Map上展示所有拥有 Location 数据的 Content 的图钉（Marker）。

### 4.2 时空交互双向定位
* **用户故事**：作为一个旅行者，我想要在点击地图图钉时自动滚动并高亮左侧时间轴的对应日记，或者点击时间轴的某天时，地图自动平移缩放到那些事件的发生地，以便于精准查看。
* **验收标准**：
  - [ ] 地图 marker 的 `bindtap` 事件能触发左侧 `scroll-into-view`。
  - [ ] 时间轴 item 的点击能触发地图 API 的 `includePoints` 或 `moveToLocation`。

### 4.3 时间地图双模式切换（当前项目 / 所有项目）
* **用户故事**：作为一个旅行者，我想要在时间地图中切换“当前项目”与“所有项目”两种模式，以便于我既能查看单次旅行的细节，也能按年份回顾全部旅行项目。
* **验收标准**：
  - [ ] 时间地图页面提供“当前项目 / 所有项目”模式切换入口。
  - [ ] 在“当前项目”模式下，时间轴展示该项目内的内容记录，地图标注该项目的全部位置点。
  - [ ] 在“所有项目”模式下，时间轴按“年”分组统计，每一项为一个旅行项目。
  - [ ] 在“所有项目”模式下，地图仍标注所有项目的内容记录点，并可与时间轴项目项进行联动定位。

## Epic 5：生成、导出与分享
**针对核心模块 4**

### 5.1 生成游记 PDF
* **用户故事**：作为一个旅行爱好者，我想要将整个旅行项目导出为A4排版的PDF文档，以便于我打印装订成纸质纪念册。
* **验收标准**：
  - [ ] 后端利用 pdfkit 等工具按时间顺序抓取图片和文字进行流式排版。
  - [ ] 提供下载链接供用户保存到本地。

### 5.2 生成静态纪念网页
* **用户故事**：作为一个旅行者，我想要一键生成当前旅行的单页网页（HTML），以便于分享微信链接给不使用该小程序的家人朋友查看。
* **验收标准**：
  - [ ] 后端渲染纯静态化网页，打包内联 CSS/JS。
  - [ ] 网页中带有简化的时间轴展示。

---

## Epic 6：隐私控制与鉴权
**针对核心模块 5**

### 6.1 隐私级别控制
* **用户故事**：作为一个注重隐私的旅行者，我想要将某些包含我家庭住址的内容设置为“仅自己可见”，或者针对特定动态进行“地理位置模糊处理”，以便于放心地将旅行日志分享给他人。
* **验收标准**：
  - [ ] Content 数据表中包含 `visibility` 字段（私密/好友/公开）。
  - [ ] 导出 PDF 或生成网页时，程序必须过滤掉非公开（或未授权）级别的数据。
