# UI 优化任务记录

> 本文件记录旅行项目管理小程序（TripTimeline）的 UI/UX 优化背景、设计决策与任务拆分，供后续开发迭代参考。

---

## 一、项目背景

**项目名称**：TripTimeline —— 旅行项目管理微信小程序（毕业设计）

**核心流程**：先创建旅行项目（project） → 再在项目下创建记录条目（entry）

**当前核心页面**：
- `pages/project-editor`：新建 / 编辑旅行项目（表单页）
- `pages/editor`：新建 / 编辑旅行记录（表单页）

---

## 二、设计语言规范

**视觉主题**："Cartographer's Journal"（测绘学家的复古日记）/ 现代实体杂志风

**色彩系统（大地的回音）**：

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景基色 Base | `#F7F5F0` | 全局页面背景，羊皮纸暖米色 |
| 主色调 Primary | `#2A4B3C` | 重要按钮、强调文字、已选标签 |
| 点缀色 Accent | `#C85A3D` | 地理标记、高亮、警告、装饰线 |
| 主文字 | `#1C1C1C` | 正文标题，近黑深木炭色 |
| 次要文字 | `#8A877E` | 标签、副标题、提示文字 |

**排版原则**：
- 大量留白，打破等宽列表惯例
- 卡片设计可不对称，参考拍立得 / 火车票风格
- 标题区使用较大字号（44rpx+），副信息用 24rpx

**动效原则**：
- 页面进入：阶梯式上浮淡入（Staggered fade-up）
- 按钮交互：`scale(0.98)` press 反馈
- 状态切换：`transition: all 0.2s`

---

## 三、设计决策记录

以下为与 AI 协作审查时确认的设计决策，与初始建议有出入的以此为准：

| 议题 | 决策 | 原因 |
|------|------|------|
| 目的地字段 | ❌ 不新增 | 融入标题即可，如"重庆一日游" |
| 旅行人数字段 | ❌ 不新增 | 当前阶段不需要 |
| 标签（旅行印记）位置 | ✅ 保持现有位置（日期后） | 用户在计划初期就会想到标签 |
| 副标题字段 | ✅ 保留 | 定位为旅行的额外补充描述 |
| 心情 / 天气字段 | ❌ 不新增 | 写入记录正文即可 |
| editor 类型切换 | ✅ 保留，但重构 | 文字为必有项（不纳入选择），移除轨迹，保留图片/音频 |
| 轨迹（GPX）功能 | ❌ 移除 | 不需要直接导入 GPX 轨迹文件 |

## 七、Task 4：`editor` UI 操作问题修复计划（2026-04-13）

> 目标：修复当前编辑页中“用户可见但结果不一致”的操作问题，优先保证数据正确性与交互可预期性。

### 4.1 问题清单与优先级

| 编号 | 问题 | 优先级 | 当前影响 |
|------|------|--------|----------|
| T4-01 | 经纬度为 `0` 时被当作空值，地图不显示、保存时可能丢失地点 | P0 | 影响地点数据正确性 |
| T4-02 | 删除图片后仍保持 `photo` 类型，保存会被“请先选择图片”拦截 | P0 | 影响提交路径，造成误操作 |
| T4-03 | 地点面板缺少手动编辑名称/地址入口，与“手动填写为主”不一致 | P1 | 影响可编辑性与文案一致性 |
| T4-04 | 隐私模式切换后摘要文案可能与真实继承策略不一致 | P1 | 影响用户判断与信任 |
| T4-05 | 录音开始/结束仅视觉禁用，缺少真实禁用态与点击拦截反馈 | P2 | 交互反馈弱，易误判卡顿 |
| T4-06 | 音频附件缺少“清空/移除”入口 | P2 | 降低可撤销性 |

| 子任务 | 内容 | 涉及页面文件 | 状态 |

| Task 4-6 | 音频区补充“移除音频”入口并回退状态 | `editor.wxml` / `editor.ts` / `editor.scss` | ✅ 已完成 |

### 4.3 验收标准（DoD）

1. 选择坐标为 `0` 的地点时，地图可见、保存后接口含有效 `location`。
2. 删除图片后可直接保存文字内容，不再触发“请先选择图片”。
3. 用户可手动修改地点名称和地址，保存后 `location_text` 与页面一致。
4. 隐私模式在 inherit/custom 切换后，摘要文案与真实策略一致。
5. 录音按钮在不可用状态下不触发误操作，并有明确视觉反馈。
6. 已选音频可一键移除，页面状态与提交数据同步清空。

### 4.4 回归测试建议

1. 新建记录：纯文字、图片、音频三路径分别保存。
2. 编辑记录：从 `photo` 切换到 `note` 并删除附件后保存。
3. 地点边界值：`lat=0`、`lon=0`、`lat/lon` 正常值三组验证。
4. 隐私：inherit/custom 循环切换后保存并重新进入页面核对。

### 4.5 执行结果记录（2026-04-13）

1. 地图与保存链路均改为“有效数值坐标”判定，`0` 坐标不再误判为空。
2. 删除图片/音频后会回退内容类型，避免无附件却被附件校验拦截。
3. 地点面板已支持手动编辑“地点名称 + 详细地址”，并与已选坐标联动。
4. 隐私模式切换时摘要文案按项目策略/自定义策略分别展示。
5. 录音开始/结束按钮在不可用状态会短路并给出明确提示。
